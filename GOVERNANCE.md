# Governance

## Maintainer

vibops-mcp is an open source project maintained by **VibOps SAS**. VibOps SAS has final authority over the project direction, roadmap, and releases.

## Decision making

- **Bug fixes and documentation** — merged by maintainers without formal process
- **New tools or breaking changes** — discussed in a GitHub issue before implementation
- **Roadmap and provider priorities** — set by VibOps SAS, informed by community feedback

## What this project is not

vibops-mcp is not governed by a foundation or a steering committee. There is no plan to transfer governance to an independent body. This ensures consistent quality and long-term alignment with the VibOps platform.

## Community

We welcome contributions, feedback, and discussion. Opening issues and pull requests is encouraged. However, VibOps SAS reserves the right to decline contributions that conflict with the product vision or quality standards.

## Versioning

vibops-mcp follows [Semantic Versioning](https://semver.org):
- **Patch** (0.1.x) — bug fixes
- **Minor** (0.x.0) — new tools, backwards compatible
- **Major** (x.0.0) — breaking changes to tool interfaces
